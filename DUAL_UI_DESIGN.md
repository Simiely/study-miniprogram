# 双布局设计规范（手机 + iPad Pro 12.9"）

> 参考项目：`word-cards`（暖色自然风儿童双语卡片）
> 适用项目：`study-miniprogram`（学习卡片小程序，数据放云存储 JSON）
> 设计原则：**iPad 端不缩放、复用手机 UI 尺寸，只重排间距与位置**

---

## 2. 双布局判定（关键约束）

微信小程序在 **iPad 上 `wx.getWindowInfo().windowWidth` 被钳制为手机比例的绘制宽度**，
导致 `<!-- @media(min-width:768px) -->` 永远不命中。因此：

- 用 `wx.getWindowInfo().screenWidth >= 768`（物理像素）判定平板
  （iPad 竖屏 768 / 横屏 1024，iPad Pro 12.9" 同此区间）
- 命中后，页面在根节点挂 `.tablet` 类：`<view class="page-root {{isTablet ? 'tablet' : ''}}">`
- **平板样式一律用 `px` 写（不用 `rpx`）**，且组件尺寸与手机端物理尺寸保持一致（不放大）

实现见 `utils/device.js → isTablet()`，三个页面 `onLoad/onShow` 注入 `isTablet` 到 `data`。

## 3. 两套布局对照

### 首页（板块列表）
| | 手机 | iPad 12.9" (`.tablet`) |
|---|---|---|
| 容器 | 单列，全宽 | `max-width:1100px` 居中 |
| 板块卡 | 竖向大卡（全宽行） | 同尺寸卡片多列网格（`width:340px`）居中，`gap:28px` |
| 卡片内 | emoji+标题+描述+计数 | 同，仅改 `flex-direction:column` 与对齐 |

→ **组件完全同尺寸，只把「单列」改为「多列居中网格」**。

### 卡片页（翻面学习）
| | 手机 | iPad 12.9" (`.tablet`) |
|---|---|---|
| 卡片 | `flex:1` 竖向填满宽度 | `width:400px` 固定（≈手机宽）居中，`height:72vh` |
| 导航 | 卡片**右侧悬浮圆形导航按钮**（竖排，和 iPad 模式一致） | 卡片**右侧竖排**（`.nav-stack` 绝对定位） |
| 顶栏 | 左返回 / 中进度 / 右统计 | 同，整体 `max-width:920px` 居中 |

→ **卡片本身不放大**，只是利用横向空间把导航挪到侧边、顶栏居中收拢。

### 统计页
| | 手机 | iPad 12.9" (`.tablet`) |
|---|---|---|
| 布局 | 单列堆叠 | 整体 `max-width:860px` 居中 |
| 概览/排行 | 同组件 | 同组件尺寸，仅栏宽收拢居中 |

→ 统计信息天然适合窄栏，平板端只需居中收窄即可，不撑大。

## 4. 为什么「不缩放、只重排」

- **缩放（rpx 整体 ×1.3）** 会让文字、圆角、点击区被等比拉大，在 12.9" 上显得空旷、边缘发虚，且违背「手机 UI 手感」。
- **复用手机尺寸 + 重排** 让用户在平板上看到的是「熟悉大小的卡片」，只是排布更舒展（多列、侧边导航、居中栏），
  既填满了大屏，又保持了一致的视觉密度与触控手感。

---

## 5. 数据与运行（不变）

- 数据仍在云存储 JSON：`boards.json` + `cards-<boardId>.json`，小程序用 `wx.cloud.downloadFile` 读取。
- 统计仍走本地 `wx.Storage`（每日 0 点快照 + 清空），**不消耗云资源**。
- 跑起来：填 `app.js` 的 `envId` → 上传 `sample-data/` 三个 JSON 到云存储根目录 → 开发者工具导入运行。

## 6. 卡片模式发音横幅交互规范

卡片学习区（卡片模式 + 浏览模式弹窗）由三块可点击发音横幅组成：

1. **中文格**（左）
2. **英文格**（右）
3. **说明行**（整行）

规范要点：

- **图片圆角 48px**（iPad 卡片模式 / 浏览弹出层均使用，与 iPhone 一致）。
- **三行发音横幅宽度与图片对齐**：`lc-texts` 左右边距为 0，横幅（`.lc-line` / `.lc-cell`）的可见背景区域与图片等宽。
- **整条横幅均可点击播放**：每个格 / 行整块为点击热区，喇叭图标仅作视觉提示，不限制点击范围（即点击文字或横幅任意处均可触发发音）。该说明仅记于本文档，不在 UI 上标注。
- **发音图标规格**：64px 主绿圆钮（`#4a7c59`），图标 32px（约为原 92/46 的 70%），整体 `opacity:0.2`（视觉弱化，不抢内容）。
- **中 / 英同排**：左 中文、右 英文 同一行两格（各自整格可点发音）；英文名称**字号、颜色与中文一致（88px / `--text`）**、靠本格**左侧**对齐，图标在本格右侧；说明文字独立一行。
- **横幅底色**：`#faf8f3`，圆角 34px，间距 22px；`:active` 态底色 `#f1ede4` + 轻微 `scale(.99)`。
