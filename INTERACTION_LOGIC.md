# 学习卡片小程序 · 交互逻辑文档

> 参考项目：`word-cards`（暖色自然风儿童双语卡片）
> UI 设计规范详见：`DUAL_UI_DESIGN.md`

---

## 1. 全局架构

### 1.1 数据源
- 卡片数据存储在**微信云存储 JSON 文件**中（无数据库）
- `boards.json` → 板块列表（ID / 标题 / 卡片数）
- `cards-<boardId>.json` → 某板块下所有卡片（含图片 URL、中文名、英文名、科普说明文、三段音频 URL）
- 小程序通过 `wx.cloud.downloadFile` 读取

### 1.2 本地存储
- **统计**走 `wx.Storage`，实时累加但不实时刷新 UI
- 每日 **0:00 自动快照**一次，显示缓存结果
- 提供「清空今日统计」按钮

### 1.3 双布局判定
参考 `utils/device.js`：

```javascript
// screenWidth >= 768 → 判定为平板（iPad）
// 注意：不能使用 windowWidth（iPad 上被钳制为手机比例）
const isTablet = wx.getWindowInfo().screenWidth >= 768;
```

- 手机：竖屏，基准布局
- 平板（iPad Pro 12.9"）：竖屏 / 横屏，组件尺寸与手机相同（不缩放），仅重排间距与位置
- 平板样式用 `px`（不用 `rpx`），根节点挂 `.tablet` 类

## 2. 两种模式

### 2.1 浏览模式

**用途**：快速浏览某板块所有卡片。

**布局**：
- 拼音排序的方阵网格
  - 手机：3 列
  - 平板竖屏：6 列
  - 平板横屏：8 列
- 格子为方形深色底 + 动物 emoji + 中文名
- 已浏览过的格子：`opacity: 0.1`（10% 透明度）+ ✓ 角标

**交互**：
| 操作 | 反馈 |
|------|------|
| 点卡片格子 | 弹出卡片详情 `pop-card`（全屏半透明蒙层 + 奶油底卡片） |
| 弹出卡片详情内 | 展示大图 + 三段发音横幅，点击 ✕ 或蒙层关闭 |
| 「再来一次」按钮 | 清除所有已浏览标记，重新开始 |

### 2.2 卡片模式

**用途**：逐个学习卡片，大图 + 发音 + 导航翻页。

**布局**（手机）：
- 顶部：模式标签 + 快速链接
- 计数器：「第 N / 20 张」
- 卡片区域：大图（尽可能大）+ 下方三段发音横幅
- 导航：右侧悬浮竖排圆形按钮（‹ / 🎲 / ›）

**布局**（平板）：
- 顶部同手机
- 卡片占满屏幕（`flip-flank` 模式）
- 三行发音横幅：浮在图片底部半透明奶油渐变条上
- 导航：右侧悬浮竖排圆形按钮（‹ / 🎲 / ›），50% 透明度

**交互**：
| 操作 | 反馈 |
|------|------|
| 点击 ‹ | 切换到上一张卡片 |
| 点击 🎲 | 随机跳到一张卡片 |
| 点击 › | 切换到下一张卡片 |
| 点击 ⤢ | 全屏查看图片（覆盖整屏，点击任意处关闭） |
| 点击发音横幅 | 播放对应语音（见第 3 节） |
| 左右滑动（平板横屏） | 翻到上一张/下一张 |

## 3. 发音交互（三种横幅）

### 3.1 结构
每张卡片含三段可发音内容：

```
┌─────────────────────────────────────┐
│              中文名（88px）    🔊    │  ← 整行可点
├─────────────────────────────────────┤
│              英文名（88px）    🔊    │  ← 整行可点
├─────────────────────────────────────┤
│       科普说明文（42px）        🔊    │  ← 整行可点
└─────────────────────────────────────┘
```

### 3.2 交互规则
- **整条横幅均可点击播放**：每个格/行为点击热区，喇叭图标仅作视觉提示，不限制点击范围
- 点击任意处（文字或横幅）触发对应语音
- 喇叭图标规格：64px 主绿圆钮 `#4a7c59`，图标 32px，整体 `opacity: 0.2`
- 横幅底色 `#faf8f3`，圆角 34px，`:active` 态变暗 `#f1ede4` + 微缩 `scale(.99)`

### 3.3 技术实现（参考）
- 使用 `wx.createInnerAudioContext()` 或 TTS 接口
- 音频 URL 从卡片数据的 `speech_zh` / `speech_en` / `speech_fact` 字段读取
- 播放期间可轻触停止并重新开始

## 4. 导航按钮

### 4.1 规格（手机 & iPad 统一）
| 属性 | 值 |
|------|-----|
| 形状 | 圆形 |
| 尺寸 | 105 × 105px |
| 图标字号 | 50px |
| 透明度 | 50%（`rgba(74,124,89,.5)`） |
| 间距 | 手机 38px / iPad 44px |
| 位置 | 卡片右侧居中偏下（手机 `top:62%` / iPad `top:60%`） |
| z-index | 10（浮在图片上方） |

### 4.2 三种按钮
| 图标 | 类名 | 功能 |
|------|------|------|
| ‹ | `.nav-round.ghost` | 上一张（不可用时置灰） |
| 🎲 | `.nav-round.accent` | 随机一张 |
| › | `.nav-round` | 下一张（不可用时置灰） |

## 5. 缩放按钮

### 5.1 位置
- 图片区域**右下角**
- 绝对定位 `bottom:30px;right:30px`
- 88px 半透明白底圆钮 + 主绿 ⤢ 图标

### 5.2 交互
| 操作 | 反馈 |
|------|------|
| 点击 ⤢ | 弹出全屏图片浮层（`.img-fs`） |
| 点击全屏浮层任意处 | 关闭浮层 |
| 浮层内 | emoji 居中放大 + 「点击任意处关闭」提示 |

## 6. 统计逻辑

### 6.1 记录
- 每次浏览一张卡片（进入卡片模式或弹出详情）→ 对应卡片浏览次数 +1
- 实时写入 `wx.Storage`（key 按日期隔离）
- 不实时刷新 UI，避免频繁 setData

### 6.2 展示
- 每日 **0:00 自动快照**：将昨日累计数据拍平为当日统计
- 统计页显示：总浏览数 / 看过卡片种类数 / 浏览排行 Top 5
- 「清空今日统计」按钮：重置当日数据（不影响历史快照）

### 6.3 交互
| 操作 | 反馈 |
|------|------|
| 进入统计页 | 显示最近一次快照的数据 |
| 点清空 | 确认弹窗 → 清空 → 刷新显示 |

## 8. 数据流与页面结构

```
app.js 初始化 envId
  │
  ├── pages/index        → 板块列表（从 boards.json 读取）
  │     └── 点板块 → pages/cards（传入 boardId）
  │
  ├── pages/cards        → 浏览模式 / 卡片模式 双模式页面
  │     ├── 浏览模式：渲染网格 → 点格子弹出详情
  │     ├── 卡片模式：大图 + 发音 + 翻页
  │     └── 统计按钮 → pages/stats
  │
  └── pages/stats        → 今日统计（从本地 Storage 读取）
        └── 清空按钮 → 重置当日数据
```

### 8.1 utils 工具模块
| 文件 | 功能 |
|------|------|
| `utils/device.js` | `isTablet()` 平板判定 |
| `utils/cloud-data.js` | `wx.cloud.downloadFile` 封装，读取 JSON |
| `utils/stats.js` | 本地统计：累加 / 快照 / 清空 |
